## Аннотация

Документ систематизирует подходы к аксиальному кодированию в рамках Grounded Theory, 
интегрируя классическую парадигму Strauss & Corbin (1990) с современными адаптациями 
Charmaz (2014) и опытом разработки многоуровневых теоретических моделей поведения. 
Представлены три стратегии аксиального кодирования с операционализированными критериями 
выбора подхода в зависимости от характера данных и исследовательских целей.

## 1. Введение

### 1.1. Место аксиального кодирования в GT

**Grounded Theory — трёхэтапный процесс:**

1. **Open Coding** — выявление концептов из данных
2. **Axial Coding** — связывание категорий через отношения
3. **Selective Coding** — интеграция в теоретическую модель

**Аксиальное кодирование = переход от "что?" к "как?" и "почему?"**

### 1.2. Проблема: множественность подходов

**Классическая парадигма** (Strauss & Corbin, 1990):
- Жёсткая структура: conditions → phenomenon → context → strategies → consequences
- Риск: навязывание априорной структуры данным

**Конструктивистская адаптация** (Charmaz, 2014):
- Гибкое выявление связей между категориями
- Риск: потеря систематичности при сложных данных

**Вопрос:** Как выбрать подход для конкретного исследования?

## 2. Три стратегии аксиального кодирования

### 2.1. СТРАТЕГИЯ 1: Парадигматическая модель (Strauss & Corbin)

#### Структура

```
УСЛОВИЯ (Causal Conditions)
    ↓
ФЕНОМЕН (Phenomenon)
    ↓
КОНТЕКСТ (Context) + ИНТЕРВЕНИРУЮЩИЕ УСЛОВИЯ (Intervening Conditions)
    ↓
СТРАТЕГИИ/ДЕЙСТВИЯ (Strategies/Actions)
    ↓
ПОСЛЕДСТВИЯ (Consequences)
```

#### Операционализация

**1. Causal Conditions** — "Что вызывает феномен?"
- Индикаторы: "потому что...", "из-за...", "причина в..."
- Пример: "Я начал флиппить, потому что увидел +50% за день"

**2. Phenomenon** — Центральная категория
- Индикаторы: повторяющееся действие/состояние в нарративах
- Пример: Flipping behaviour

**3. Context** — "При каких условиях феномен проявляется?"
- Индикаторы: "в ситуации когда...", "если...", "при..."
- Пример: "при высокой волатильности IPO"

**4. Intervening Conditions** — "Что модифицирует связь условия→действие?"
- Индикаторы: "но...", "однако...", "несмотря на..."
- Пример: "но у меня было правило не трогать 5 лет"

**5. Strategies/Actions** — "Что делает агент в ответ?"
- Индикаторы: глаголы действия
- Пример: "продал половину", "держал позицию"

**6. Consequences** — "К чему приводят действия?"
- Индикаторы: "в результате...", "после этого..."
- Пример: "в результате зафиксировал прибыль, но потерял дальнейший рост"

#### Когда использовать

✅ **Подходит когда:**
- Исследуется процесс с чёткой причинно-следственной структурой
- Данные содержат явные последовательности событий
- Цель: построить процессуальную модель (flowchart-like)

❌ **НЕ подходит когда:**
- Феномен сложный, с множественными одновременными влияниями
- Данные фрагментарны, нет чётких каузальных цепочек
- Категории формируются индуктивно и не укладываются в парадигму

#### Пример применения

**Тема:** Импульсивное поведение при IPO

```
CAUSAL CONDITIONS:
- Price surge (+20-50% за день)
- FOMO (fear of missing out)

PHENOMENON:
- Impulsive flipping (продажа вопреки стратегии)

CONTEXT:
- First-time IPO investor
- High volatility market
- No prior rules

INTERVENING CONDITIONS:
- Presence of commitment mechanisms → блокирует импульс
- High inhibitory control → задерживает действие

STRATEGIES:
- Immediate sell (импульсивное действие)
- Delayed sell (волевое усилие → истощение)
- Compromise sell (partial)

CONSEQUENCES:
- Material: profit/loss
- Psychological: cognitive dissonance
- Belief updating: "я не дисциплинирован"
```

### 2.2. СТРАТЕГИЯ 2: Конструктивистская (Charmaz)

#### Структура

**Нет фиксированной структуры.** Исследователь выявляет связи через:

1. **Сравнение категорий** — "Как X связан с Y?"
2. **Поиск процессов** — "Как X развивается во времени?"
3. **Выявление действий** — "Что делают агенты с X?"

#### Операционализация через вопросы

**Базовые вопросы:**
- Когда, почему и как возникает категория X?
- Что способствует/препятствует X?
- Что меняется при изменении X?
- Каковы последствия X?

**Специфические вопросы:**
- Как агенты интерпретируют X? (subjective meaning)
- Как X встроен в социальный контекст?
- Как X влияет на идентичность агента?

#### Когда использовать

✅ **Подходит когда:**
- Исследуется субъективный опыт и смыслы
- Категории возникают индуктивно и не укладываются в линейную модель
- Цель: построить концептуальную сеть (network-like)

❌ **НЕ подходит когда:**
- Нужна строгая формализация для количественной валидации
- Данные требуют систематической структуры для операционализации

#### Пример применения

**Тема:** Identity work при конфликте целей

**Вопросы для аксиального кодирования:**
- Как инвесторы интерпретируют флиппинг? → "не по мне", "потеря контроля"
- Как флиппинг влияет на идентичность? → когнитивный диссонанс
- Какие стратегии используют для защиты идентичности? → рационализация

**Выявленные связи:**
```
Strategic Goal (identity) → Conflict Detection → Ego Depletion → Flipping 
    ↓
Cognitive Dissonance → Rationalization → Identity Protection
```

### 2.3. СТРАТЕГИЯ 3: Гибридная (интеграция парадигмы + конструктивизма)

#### Структура

**Базовая парадигма** (для процессуальной основы):
```
Conditions → Phenomenon → Actions → Consequences
```

**+ Конструктивистские расширения** (для субъективности):
- Subjective interpretation слоя
- Identity work как meta-процесс
- Social context как модулятор

#### Операционализация

**ШАГ 1: Построить базовую парадигматическую модель**
- Определить phenomenon, conditions, actions, consequences

**ШАГ 2: Добавить конструктивистские измерения**
- Как агент интерпретирует conditions? (subjective layer)
- Как actions влияют на identity? (meta-consequence)
- Как social context модифицирует process?

**ШАГ 3: Интегрировать через мемо**
- Написать теоретическое мемо о связях между слоями

#### Когда использовать

✅ **Подходит когда:**
- Нужна формальная модель (для операционализации) + интерпретативная глубина
- Данные содержат процессуальные и субъективные компоненты
- Цель: построить многоуровневую теоретическую модель

❌ **НЕ подходит когда:**
- Ограниченные данные (нет ресурсов на два прохода кодирования)
- Простой феномен, не требующий многослойности

#### Пример применения (наш опыт)

**Волевая архитектура поведения:**

**Парадигматический слой (процессуальная структура):**
```
TRIGGER (price surge) 
    ↓
IMPULSE ACTIVATION 
    ↓
CONFLICT DETECTION 
    ↓
VOLITIONAL PROCESS (arbitration) 
    ↓
ACTION (hold/flip/compromise)
    ↓
CONSEQUENCES (material + psychological)
```

**Конструктивистский слой (субъективность):**
- **Identity:** "Я пассивный инвестор" → conflict intensification
- **Interpretation:** "Это упущенная возможность" vs "Это краткосрочный шум"
- **Meta-reflection:** "Я потерял контроль" → identity threat

**Интеграция:**
- Identity strength → модифицирует volitional strength
- Interpretation → модифицирует salience of impulse
- Meta-reflection → запускает rationalization

## 3. Критерии выбора стратегии

### 3.1. Матрица решений

| Критерий | Парадигматическая | Конструктивистская | Гибридная |
|----------|-------------------|-------------------|-----------||
| **Характер данных** | Процессуальные нарративы | Рефлексивные интервью | Оба типа |
| **Структурированность феномена** | Высокая (чёткие этапы) | Низкая (размытые границы) | Средняя |
| **Цель исследования** | Операционализация + формализация | Концептуальная сеть смыслов | Многоуровневая модель |
| **Тип модели** | Flowchart / Process model | Network / Conceptual map | Layered architecture |
| **Сложность анализа** | Средняя | Низкая | Высокая |
| **Риск навязывания структуры** | Высокий | Низкий | Средний |

### 3.2. Эвристика выбора

**ШАГ 1: Проверить характер феномена**

```python
if phenomenon_has_clear_stages AND causal_chains_visible:
    consider_paradigmatic()
elif phenomenon_is_interpretative AND meanings_central:
    consider_constructivist()
else:
    consider_hybrid()
```

**ШАГ 2: Проверить цель исследования**

```python
if goal == "quantitative_validation_later":
    prefer_paradigmatic_or_hybrid()
elif goal == "interpretative_theory":
    prefer_constructivist()
elif goal == "multilevel_model":
    prefer_hybrid()
```

**ШАГ 3: Проверить ресурсы**

```python
if data_is_rich AND time_available:
    hybrid_possible = True
else:
    choose_simpler_approach()
```

## 4. Практические техники аксиального кодирования

### 4.1. Техника 1: Вопросы к данным (Question Prompts)

**Для парадигматической модели:**

1. **Conditions:** "Что предшествует X?" "Почему X возникает?"
2. **Context:** "При каких обстоятельствах X проявляется?"
3. **Actions:** "Что делают агенты в ответ на X?"
4. **Consequences:** "К чему приводит X?"

**Для конструктивистской модели:**

1. "Как агенты придают смысл X?"
2. "Как X связан с идентичностью?"
3. "Как меняется X во времени?"
4. "Какие социальные факторы влияют на X?"

### 4.2. Техника 2: Диаграммы связей (Relationship Mapping)

**Шаг 1: Список всех категорий из open coding**

```
- Strategic goal
- Tactical impulse
- Conflict
- Inhibitory control
- Commitment mechanism
- Flipping
- Rationalization
```

**Шаг 2: Для каждой пары категорий — спросить:**
- Есть ли связь?
- Какого типа связь? (causal / conditional / temporal / interpretative)
- Какова направленность?

**Шаг 3: Построить диаграмму**

```
Strategic Goal --[conflicts with]--> Tactical Impulse
    ↓
Conflict --[activates]--> Inhibitory Control
    ↓
Inhibitory Control --[modulated by]--> Commitment Mechanism
    ↓
Control Outcome --[determines]--> Action (hold/flip)
    ↓
Action --[triggers]--> Rationalization (if conflict with identity)
```

### 4.3. Техника 3: Теоретическое мемо (Theoretical Memoing)

**Структура мемо:**

```markdown
## Memo: Связь между Conflict и Ego Depletion

**Дата:** 2025-12-27
**Категории:** Conflict, Ego Depletion, Inhibitory Control

**Наблюдение:**
Респонденты, вербализующие высокий конфликт ("разрывался", "не знал что делать"), 
часто описывают последующую потерю контроля ("не выдержал", "устал держаться").

**Предположение:**
Интенсивность конфликта → расход ресурса самоконтроля (ego depletion) → 
снижение inhibitory control → импульсивное действие.

**Связь с литературой:**
Baumeister (2007): самоконтроль = истощаемый ресурс. Последовательные акты 
самоконтроля истощают ресурс.

**Testable prediction:**
Временная задержка действия (индикатор волевого усилия) должна коррелировать 
с вербализацией истощения.

**Следующий шаг:**
Кодировать все случаи вербализации временной задержки + вербализации истощения.
Подсчитать частоту совместной встречаемости.
```

### 4.4. Техника 4: Сравнительный анализ случаев (Constant Comparison)

**Цель:** Уточнить условия связи между категориями через сравнение случаев.

**Пример:**

**Вопрос:** При каких условиях commitment mechanism блокирует импульс?

**Случай 1:** "У меня правило не трогать 5 лет" → Держал позицию при +50%
- Commitment: Explicit rule
- Activation: High impulse
- Outcome: Blocked

**Случай 2:** "Я думал держать долго, но..." → Продал при +30%
- Commitment: Implicit intention (weak)
- Activation: Medium impulse
- Outcome: Failed

**Вывод:** Commitment mechanism эффективен, если:
1. Explicit (вербализованное правило)
2. Pre-established (установлено до триггера)
3. Linked to meta-goal ("если нарушу, потеряю дисциплину")

## 5. Валидация аксиального кодирования

### 5.1. Критерии качества связей

**1. Empirical Grounding** — связь подтверждена данными
- Индикатор: минимум 3 случая поддерживают связь
- Проверка: поиск deviant cases (случаев, где связь не работает)

**2. Conceptual Clarity** — связь концептуально чёткая
- Индикатор: можно сформулировать в одном предложении
- Проверка: коллега понимает связь без дополнительных объяснений

**3. Theoretical Coherence** — связь согласуется с другими связями
- Индикатор: нет противоречий в модели
- Проверка: построение диаграммы выявляет логические петли

**4. Explanatory Power** — связь объясняет вариативность
- Индикатор: связь помогает различать случаи
- Проверка: условия связи объясняют, когда она работает/не работает

### 5.2. Техника: Negative Case Analysis

**Цель:** Уточнить границы применимости связи.

**Процесс:**
1. Сформулировать связь: "Commitment mechanism блокирует импульс"
2. Найти deviant case: "Было правило, но всё равно продал"
3. Проанализировать: "Почему связь не сработала?"
4. Уточнить связь: "Commitment mechanism блокирует импульс, ЕСЛИ:
   - Explicit
   - High salience в момент триггера
   - Не истощён ресурс самоконтроля"

## 6. Переход к селективному кодированию

### 6.1. Когда завершать аксиальное кодирование?

**Критерии насыщения:**

1. **Theoretical Saturation** — новые данные не добавляют новых связей
2. **Model Coherence** — все категории интегрированы в связную модель
3. **Explanatory Coverage** — модель объясняет вариативность в данных

### 6.2. Подготовка к selective coding

**Итог аксиального кодирования:**
- Диаграмма связей между всеми категориями
- Набор теоретических мемо о ключевых связях
- Список условий для каждой связи

**Переход к selective coding:**
- Выбор core category (центральной категории)
- Интеграция всех связей вокруг core category
- Написание storyline (теоретического нарратива)

## 7. Пример: Аксиальное кодирование волевой архитектуры

### 7.1. Применённая стратегия: Гибридная

**Парадигматический слой:**

```
TRIGGER (price surge)
    ↓
IMPULSE ACTIVATION (System 1: "grab profit")
    ↓
CONFLICT DETECTION (ACC: incompatibility detected)
    ↓
[VOLITIONAL PROCESS] ← CORE PHENOMENON
    ├─ Goal strength evaluation
    ├─ Inhibitory control deployment
    ├─ Commitment mechanism check
    ↓
ACTION (hold/flip/compromise)
    ↓
CONSEQUENCES
    ├─ Material (profit/loss)
    ├─ Psychological (dissonance)
    └─ Reflection (rationalization)
```

**Конструктивистский слой:**

- **Identity strength** → модифицирует goal strength
- **Subjective interpretation** of trigger → модифицирует salience
- **Meta-reflection** on action → запускает rationalization

### 7.2. Ключевые связи (с мемо)

**Связь 1: Conflict → Ego Depletion**
- Тип: Causal
- Условия: High conflict intensity + Temporal delay
- Механизм: Волевое усилие истощает ресурс
- Индикаторы: "устал держаться", "не выдержал"

**Связь 2: Commitment Mechanism → Impulse Blocking**
- Тип: Conditional
- Условия: Explicit + Pre-established + High salience
- Механизм: Pre-commitment автоматически блокирует действие
- Индикаторы: "у меня правило", "я себе обещал"

**Связь 3: Identity Threat → Rationalization**
- Тип: Interpretative
- Условия: Action conflicts with identity
- Механизм: Cognitive dissonance reduction
- Индикаторы: "хотя понимаю что...", "но зато..."

## 8. Заключение

### 8.1. Рекомендации по выбору стратегии

**Для начинающих:**
- Начать с конструктивистской стратегии (меньше риска навязывания структуры)
- По мере формирования категорий — добавить парадигматические элементы

**Для опытных:**
- Использовать гибридную стратегию при сложных феноменах
- Начинать с парадигмы → добавлять конструктивистские измерения

### 8.2. Критические замечания

**Риск овер-интерпретации:**
- Аксиальное кодирование может создать иллюзию каузальных связей там, где их нет
- Митигация: Negative case analysis + Peer debriefing

**Риск навязывания структуры:**
- Парадигматическая модель может заставить данные "уложиться" в схему
- Митигация: Проверять альтернативные конфигурации связей

**Риск потери оригинальности:**
- Слишком сильная опора на литературу может подавить ground-up теорию
- Митигация: Сначала кодировать → потом искать в литературе
